## ✅ PowerShell용 (복붙)

> 먼저 PySide6 플러그인 경로를 변수로 잡은 뒤 빌드합니다.

```powershell
# 1) PySide6 플러그인 경로 자동 탐색 (경로 출력 확인)
python -c "import PySide6, pathlib; print((pathlib.Path(PySide6.__file__).parent / 'plugins'))"

# 2) 위에서 출력된 경로를 $qt 변수에 복사해 넣으세요. 예시:
$qt = "$env:LOCALAPPDATA\Programs\Python\Python313\Lib\site-packages\PySide6\plugins"

# 3) 빌드
pyinstaller .\ScentSmart.py `
  -n ScentSmart `
  --noconfirm --clean `
  --onedir --windowed `
  --add-data "ui;ui" `
  --collect-all PySide6 `
  --add-data "$qt\platforms;PySide6\plugins\platforms" `
  --add-data "$qt\imageformats;PySide6\plugins\imageformats" `
  --add-data "$qt\styles;PySide6\plugins\styles"
```

## ✅ CMD(명령 프롬프트)용 (복붙)

```cmd
REM 1) 플러그인 경로 확인 (출력값 확인)
python -c "import PySide6, pathlib; print((pathlib.Path(PySide6.__file__).parent / 'plugins'))"

REM 2) 위에서 출력된 경로를 아래 %QT%에 맞게 넣으세요. 예시:
set "QT=%LOCALAPPDATA%\Programs\Python\Python313\Lib\site-packages\PySide6\plugins"

REM 3) 빌드
pyinstaller ScentSmart.py ^
  -n ScentSmart ^
  --noconfirm --clean ^
  --onedir --windowed ^
  --add-data "ui;ui" ^
  --collect-all PySide6 ^
  --add-data "%QT%\platforms;PySide6\plugins\platforms" ^
  --add-data "%QT%\imageformats;PySide6\plugins\imageformats" ^
  --add-data "%QT%\styles;PySide6\plugins\styles"
```

---

### 왜 이렇게 하면 되나요?

- `--collect-all PySide6`로 PySide6의 모듈/데이터를 광범위하게 수집하고,
    
- **Qt가 꼭 필요로 하는 플러그인 폴더**(platforms, imageformats, styles)를 `--add-data`로 **강제로 포함**시킵니다.
    
- 이렇게 하면 `Qt platform plugin could not be initialized` 류 오류를 피하면서, `ui/` 폴더도 함께 들어가 **코드 수정 없이** 실행됩니다.
    

### 빌드 후 체크

- `dist\ScentSmart\` 폴더 안에 `ScentSmart.exe`와 `ui\` 폴더, 그리고 `PySide6\plugins\platforms` 등이 들어가 있어야 정상입니다.
    
- 실행 시 문제 생기면 콘솔 메시지 알려주세요. (다른 리소스 폴더가 더 있으면 같은 방식으로 `--add-data "폴더;폴더"` 추가하면 됩니다.)
