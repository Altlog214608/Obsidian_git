GitHub Copilot

아래는 프로젝트 루트에 흔히 있는 파일/폴더들의 용도와 이 코드베이스에서 우선적으로 확인해야 할 항목들을 간단하게 정리한 것입니다. (작업 폴더의 실제 파일 목록을 제가 직접 읽을 수는 없으니, 일반적인 구성과 이 레포에서 중요도가 높은 항목 위주로 정리했습니다.)

핵심 폴더
- src/  
  - 앱 소스(이미 제공). main (Electron main), preload (브리지), renderer (React UI).
- resources/  
  - 런타임에 앱이 참조하는 정적 리소스(이미지, 로컬 JSON, 번들 리소스). 현재 비어있을 수 있음.
- public/  
  - 정적 웹 자원(HTML 템플릿, 정적 파일) — renderer가 로드.

루트 주요 파일들 (설명)
- package.json  
  - 프로젝트 메타정보, 의존성, 스크립트(예: dev, build, start, package). 가장 먼저 열어볼 파일.
- package-lock.json / yarn.lock / pnpm-lock.yaml  
  - 설치된 의존성의 고정(재현 가능한 설치).
- tsconfig.json  
  - TypeScript 컴파일러 설정(타겟, 모듈, path alias, resolveJsonModule 등). 타입 관련 이슈 해결 시 필수 확인.
- vite.config.ts (또는 webpack.config.js)  
  - 개발 서버/번들러 설정(Vite 사용 시). 빌드 경로, 플러그인, alias, 리졸브 규칙 등.
- electron-builder config / build config (package.json 내 또는 별도 yml/toml)  
  - 패키징/배포 설정(아이콘, 플랫폼, 설치 파일 포맷).
- .env / .env.*  
  - 환경변수(개발/프로덕션 키, API 토큰 등). 민감정보 유무 체크 필요.
- .npmrc  
  - npm 동작 설정(레지스트리, 캐시, auth 등).
- .gitignore  
  - Git에 포함하지 않을 파일 목록(빌드 결과, node_modules 등).
- README.md  
  - 프로젝트 설명, 설치/실행 방법, 개발 노트.
- .eslintrc(.json/.js) / .prettierrc  
  - 코드 스타일/정적분석 규칙.
- .github/workflows/*.yml  
  - CI/CD 설정(GitHub Actions). 자동 빌드/테스트/릴리즈 정의.
- dockerfile / docker-compose.yml  
  - 도커 관련 설정(있다면).
- *.yml / *.yaml / *.toml (임의 구성파일)  
  - CI, 릴리즈, 래퍼(전자 서명 등) 또는 외부 도구 설정에 사용.
- README/CONTRIBUTING/LICENSE 등 문서 파일  
  - 협업/라이선스 정보.

프로젝트 내부(렌더러/메인)에서 중요한 설정 파일
- env.d.ts / types 관련 파일  
  - 윈도우 전역 객체 타입(예: preload로 노출한 electron API 타입) 선언. 타입스크립트 안전성 관련.
- vite/webpack 플러그인 설정(이미지, SVG, CSS 처리)  
  - 로더/플러그인이 자원 로딩에 영향. 빌드 실패 시 여기부터 점검.

우선적으로 열어볼 파일(코드 리뷰 우선순위)
1. package.json — scripts, dependencies, electron 관련 스크립트 확인  
2. vite.config.ts — 개발 URL, build output, alias 확인  
3. tsconfig.json — 경로 alias, JSX 설정, strict 옵션  
4. index.ts — Electron 창 생성, preload 경로, IPC 채널  
5. index.ts (+ index.d.ts) — 렌더러에 노출된 API 목록/타입  
6. main.tsx & App.tsx — React 진입점과 라우팅 구조  
7. src/renderer/src/pages/* — 유저 플로우(홈, 커스텀, 결과 등)  
8. tracking/*, session.ts — 분석/로그 초기화(외부 키 토큰 존재 여부 확인)  
9. public/ & resources/ — 정적 자원 누락 여부 확인

빠른 파일 목록 출력(Windows PowerShell)
- 프로젝트 루트에서 파일/폴더 전체 구조 확인:
  - PowerShell:
    powershell -Command "Get-ChildItem -Recurse -Force | Where-Object { -not $_.PSIsContainer } | Select-Object FullName"
  - 간단한 트리(명령 프롬프트):
    tree /F

유의사항 & 체크리스트 (인수 전 반드시 확인)
- package.json 스크립트로 dev/build/start가 정상 동작하는지 확인. (npm ci → npm run dev)  
- .env 파일에 민감정보(토큰)이 하드코딩되어 있지 않은지 확인.  
- preload에서 노출한 API와 renderer에서 사용하는 채널 이름이 일치하는지 확인.  
- vite/tsconfig의 별칭(alias)이 코드 import와 일치하는지 확인(빌드 시 모듈 미발견 오류 빈발).  
- resources/public에 있어야 하는 외부 리소스(이미지, JSON 등)가 누락되면 런타임 에러 발생.  
- CI 설정(.github/workflows)과 빌드 스크립트를 읽어 패키징 절차를 이해.
