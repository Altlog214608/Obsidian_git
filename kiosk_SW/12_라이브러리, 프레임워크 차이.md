# 프레임워크와 라이브러리의 차이

## 핵심 차이점
- **제어 흐름 (Control Flow)**
  - 라이브러리: 개발자가 흐름 제어
  - 프레임워크: 프레임워크가 흐름 제어 ("IoC - Inversion of Control")

## 예시로 보는 차이 (현재 프로젝트 기준)

### 1. Electron (프레임워크)
```typescript
// Electron이 라이프사이클을 제어
app.whenReady().then(() => {
  createWindow()
})

// Electron이 이벤트를 제어
app.on('window-all-closed', () => {
  if (process.platform !== 'darwin') {
    app.quit()
  }
})
```

### 2. React (프레임워크)
```typescript
// React가 컴포넌트 라이프사이클과 렌더링을 제어
function App() {
  useEffect(() => {
    // 개발자는 훅을 통해 React의 라이프사이클에 "참여"
  }, [])

  return <div>...</div>
}
```

### 3. D3 (라이브러리)
```typescript
// 개발자가 D3 메서드를 원하는 시점에 호출
import { select } from 'd3'

// 개발자가 흐름 제어
const svg = select('#chart')
  .append('svg')
  .attr('width', 100)
  .attr('height', 100)
```

## 주요 특징 비교

### 프레임워크
- 애플리케이션의 구조와 흐름을 정의
- 개발자는 프레임워크가 정한 규칙을 따라야 함
- 예: Electron, React, Angular

### 라이브러리
- 특정 기능을 제공하는 도구 모음
- 개발자가 필요한 시점에 호출해서 사용
- 예: D3, GSAP, Recharts

## 현재 프로젝트의 구성
- **프레임워크**: Electron, React (구조/흐름 제어)
- **라이브러리**: D3, GSAP, Recharts (기능 제공)

이해를 돕기 위해 "Hollywood Principle"이라는 비유를 사용합니다:
- 프레임워크: "Don't call us, we'll call you" (할리우드 원칙)
- 라이브러리: "I'll call you when I need you" (개발자가 필요할 때 호출)

# 프레임워크와 라이브러리 관계 설명

맞습니다. 현재 열려있는 `index.ts` 파일을 예시로 설명하겠습니다.

## 프레임워크 예시 (Electron)
Electron은 프레임워크로서 자체적인 규칙과 구조를 가지고 있습니다:

1. 생명주기 제어
````typescript
// Electron이 정한 방식대로 앱 초기화를 해야 함
app.whenReady().then(() => {
  createWindow()
})
````

2. 윈도우 생성 규칙
````typescript
// Electron의 BrowserWindow 클래스를 사용해야만 창을 만들 수 있음
const mainWindow = new BrowserWindow({
  width: Math.floor(width),
  height: Math.floor(height),
  webPreferences: {
    preload: join(__dirname, '../preload/index.js'),
    sandbox: false
  }
})
````

## 라이브러리 예시 (path, electron-toolkit/utils)
라이브러리는 필요한 기능만 가져다 사용할 수 있습니다:

1. path 라이브러리
````typescript
import { join } from 'path'  // path 라이브러리에서 join 함수만 가져와서 사용
````

2. electron-toolkit/utils 라이브러리
````typescript
import { electronApp, optimizer, is } from '@electron-toolkit/utils'  // 필요한 유틸리티 함수만 선택적으로 사용
````

## 차이점 정리
1. **프레임워크 (Electron)**
   - 앱의 구조를 정의
   - 자체적인 규칙이 있음 (app.whenReady(), BrowserWindow 사용 등)
   - 개발자는 프레임워크의 규칙을 따라야 함

2. **라이브러리 (path, utils 등)**
   - 필요한 기능만 선택해서 사용
   - 사용 방식이 자유로움
   - 개발자가 원하는 시점에 호출

즉, 프레임워크는 "집"이고 라이브러리는 "가구"라고 비유할 수 있습니다:
- 프레임워크: 집의 구조와 규칙이 이미 정해져 있음
- 라이브러리: 필요한 가구를 선택해서 집 안에 배치할 수 있음
